## Context

Partial Machine analysis currently builds a Hann-windowed mono frame, transforms it to a DFT, extracts prominent local maxima as atoms, tracks them with attack/decay/portamento dynamics, then resynthesizes only those atoms into a quad OLA frame. Broadband or non-peaked spectral energy that remains after the atom model is not represented in the tracked state or the synthesized return.

`SpectralModelGeneric` is the right owner for residual analysis state because it already owns DFT analysis, atom extraction, parameter indexing, and attack/decay smoothing. `PartialMachine` is the right owner for residual quad synthesis because it already owns reduction, pan/radius/azimuth mapping, random helpers, `QuadDFT`, and `QuadOLA`.

## Goals / Non-Goals

**Goals:**

- Represent residual spectral energy as a queryable envelope with one bucket per DFT component inside `SpectralModelGeneric`.
- Compute analysis residuals by canceling extracted input partials from the analysis DFT at opposite phase, then measuring the magnitudes that remain.
- Smooth residual bucket magnitudes with the existing frequency-dependent attack and decay parameters.
- Synthesize residual energy into target quad DFT buckets by applying envelope, reduction, pan, and random phase, then adding complex values directly to the target components.
- Apply reduction feedback back into residual model bucket magnitudes, matching atom synthesis feedback behavior.
- Keep the feature realtime-safe with fixed arrays and no per-hop allocation.

**Non-Goals:**

- Do not add new UI controls, preset schema fields, or visualization requirements in this change.
- Do not change the existing atom tracking contract except where analysis exposes residual magnitudes.
- Do not introduce a separate residual OLA path; residuals are part of the same synthesis frame as atoms.

## Decisions

### Use one residual envelope bucket per DFT component

Add `ResidualModel` inside `SpectralModelGeneric` with:

- `static constexpr size_t x_numBuckets = DFT::x_maxComponents`, so every positive-frequency DFT component has a matching residual envelope bucket.
- `float m_magnitudes[x_numBuckets]` for the smoothed residual envelope.
- a fixed array of normalized linear bucket frequencies in cycles per sample, where bucket `k` maps to DFT component `k`.
- a fixed precomputed array of log-frequency coordinates for those same DFT buckets. This array exists as an optimization for frequency-dependent parameter indexing; processing should use it through `GetIndexForLogFrequency` so it does not take a logarithmic conversion every hop.
- an `Input` struct with `float m_analysisResidualMagnitudes[x_numBuckets]`.
- a `GetEnvelope(size_t bucketIndex)` query that returns `m_magnitudes[bucketIndex]` for the matching DFT bucket.

Each residual bucket maps to a frequency-dependent parameter index using its precomputed log-frequency coordinate and the current `spectralInput.m_parameterInput`. `ResidualModel::Process(SpectralModel::Input& spectralInput, ResidualModel::Input& residualInput)` folds the input magnitude into the stored magnitude with `BiDirectionalSlew::Process`, using `spectralInput.m_slewUpAlpha` and `spectralInput.m_slewDownAlpha` at that bucket's parameter index.

Alternative considered: store a lower-resolution envelope at `DFT::x_maxComponents / 4` or `DFT::x_maxComponents / 8`. That would save some residual model state, but it introduces aggregation and envelope-estimation choices that are not needed here. One bucket per DFT component makes analysis residual extraction and quad DFT synthesis direct: the spectral envelope at component `k` is exactly the smoothed residual magnitude at index `k`.

Alternative considered: create residual atoms with random phases and feed them through normal atom tracking. That would reuse more synthesis code but would blur the distinction between stable partials and residual noise, and would make the atom cap and visualization state represent noise buckets as partials.

### Derive residual analysis after atom extraction

`AnalysisAtom` should gain an `m_analysisPhase` field filled during peak extraction from the source DFT at the refined peak frequency. `ExtractAtoms` should keep the existing DFT used for peak finding, but after extracting the organic analysis atoms it should synthesize those atoms back into that DFT at opposite phase using the stored analysis phase. The cancellation write should use the same windowed-partial kernel family as atom synthesis so the subtraction matches the peak model closely enough to leave broadband residual energy instead of modeled partial energy.

Synthetic harmonics must not be canceled from the input residual, because they are generated by the model rather than present as original input energy. If implementation keeps synthetic harmonics in the same analysis array, residual cancellation must skip `m_isSynthetic` atoms.

After cancellation, residual analysis should copy the remaining DFT magnitudes directly into `ResidualModel::Input::m_analysisResidualMagnitudes`, one magnitude per DFT component. No averaging or cepstral liftering is needed in this version because the residual model has the same bucket count as the DFT.

### Extend spectral processing without changing existing callers abruptly

Add an overload or replacement process path that accepts both `SpectralModel::Input` and `ResidualModel::Input`. `PartialMachine::Process` should call this path on each hop after constructing the windowed buffer. Existing non-residual callers such as Deep Vocoder should either continue using the existing `ExtractAtoms(buffer, input)` entry point or pass a temporary residual input that is ignored, depending on which keeps the generic API clearest.

The residual model should live as a member beside `m_atoms`, so `PartialMachine` can read the smoothed residual magnitudes after spectral processing without duplicating state.

### Add ResidualMachine inside PartialMachine

Add `ResidualMachine` inside `PartialMachine`. For each residual bucket it should:

- select a target DFT bucket index and read `m_spectralModel.m_residualModel.GetEnvelope(bucketIndex)`;
- compute the frequency-dependent index for that bucket frequency using the same spectral parameter input as atom tracking;
- compute reduction, radius, azimuth, and quad distribution using `SynthesisContext` helpers;
- compute per-channel magnitude as `residualEnvelope * reduction * pan[channel]`;
- choose a random phase for the target bucket for the current synthesis frame;
- create a complex value at that magnitude and random phase for each channel;
- add that complex value directly into the target component of the same `QuadDFT` used for atom synthesis, preserving whatever partial energy was already there.
- compute a feedback-shaped residual magnitude using the same reduction-feedback parameter as atom synthesis;
- write that feedback-shaped magnitude back into `m_spectralModel.m_residualModel.m_magnitudes[bucketIndex]`.

This path intentionally does not call `WriteWindowedPartial` for residual synthesis. The residual model already represents a bucketed envelope, so synthesis writes directly to target DFT components instead of spreading each residual bucket with another partial kernel. Using the same `QuadDFT` still ensures partials and residuals share one inverse transform and one `QuadOLA::Write` call per hop. `ResidualMachine` should use existing random infrastructure such as `RGen` instead of adding a dependency.

Reduction feedback should mirror `SynthesisContext::ProcessAtom`: the residual bucket's stored magnitude is interpolated toward its reduced magnitude according to `m_reductionFeedback.ProcessLinear(index)`. The writeback target should use the same death-magnitude floor policy as atom reduction feedback so residual feedback and partial feedback erode spectral content consistently.

## Risks / Trade-offs

- Residual cancellation may under-cancel or over-cancel if stored analysis phase does not match the refined atom peak. Mitigation: compute and store atom analysis phase during peak extraction, then use that phase for opposite-phase cancellation.
- Random residual phases can make tests flaky if they assert exact samples. Mitigation: test finite/bounded output and deterministic state transitions where possible, and reset existing random test fixtures before DSP tests.
- One residual bucket per DFT component costs more state and per-hop work than a downsampled envelope. Mitigation: the arrays are fixed-size and allocation-free, and the simpler one-to-one mapping removes aggregation work and implementation ambiguity.
- Reduction feedback mutates residual model state during synthesis, so processing order matters. Mitigation: run residual synthesis once per hop in the same synthesis frame as atoms and write back only after computing that bucket's output magnitude for the frame.
- Sharing attack/decay with atoms couples partial and residual dynamics. Mitigation: this is intentional for the first change because the request calls for frequency-dependent attack and decay parameters; separate residual timing controls can be proposed later if needed.
